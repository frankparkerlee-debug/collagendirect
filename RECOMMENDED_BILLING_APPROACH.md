## What We Changed for Compliance

### **Fix #1: MDM-Based Documentation (Not Time-Based)**

**BEFORE (Risky):**
```
Clinical note: "Reviewed wound photo. Improving. Time: 3 minutes"
Billing: 99213 ($92)
```
❌ **Problem:** 99213 requires 20-29 minutes, but documented 3 minutes = audit red flag

**AFTER (Compliant):**
```
MEDICAL DECISION MAKING:
Problems: Diabetic foot ulcer (high complexity - chronic with complications)
Data Reviewed: Current wound photo, compared to 3 prior images from 10/15, 10/22, 10/29
Risk: Moderate - managing prescription wound care, infection risk

Assessment: Wound healing progressing well. Continue current treatment plan.
Plan: Continue collagen dressing 4x/week, reassess in 7 days

MDM Complexity: Moderate → CPT 99214 ($130)
```
✅ **Solution:** No time mentioned. Billed on clinical complexity (MDM), which takes 5-10 minutes to document

**This is already auto-generated by your system's `generateClinicalNote()` function!**

### **Fix #2: Removed Modifier 95**

**BEFORE:**
```
CPT: 99214-95
Modifier 95 = "Synchronous telehealth via interactive audio and video"
```
❌ **Problem:** Wound photo reviews are asynchronous (not real-time), so modifier 95 is incorrect

**AFTER:**
```
CPT: 99214
Modifier: (none)
Place of Service: 02 (Telehealth)
```
✅ **Solution:** No modifier claim = no synchronous vs asynchronous confusion

**Note:** Some payers may require modifier 95 even for store-and-forward. Check your contracts and add it back selectively if needed.

### **Fix #3: Same-Day Duplicate Prevention**

**BEFORE:**
- Doctor could review 3 photos from same patient in one day
- Bill 99213 × 3 = $276

❌ **Problem:** Medicare allows only 1 E/M per patient per day per provider

**AFTER:**
- System blocks second review attempt same day
- Error: "Cannot bill multiple E/M encounters for same patient on same day"

✅ **Solution:** Compliant with Medicare frequency rules

### **Fix #4: Credential-Based Reimbursement**

**BEFORE:**
- All providers billed at same rate ($92, $130, $180)

❌ **Problem:** NPs and PAs typically get 85% reimbursement, not 100%

**AFTER:**
- MD/DO: 100% rate → $92, $130, $180
- NP/PA: 85% rate → $78.20, $110.50, $153.00
- RN: Blocked from billing E/M codes

✅ **Solution:** Accurate reimbursement calculations, prevents RN billing errors

## Common Questions

### Q: "What if a patient sends multiple photos in one day?"

**A:** Doctor can review all photos, but only bill ONE encounter per day.

**Workaround:**
- Review all photos together in one assessment
- Clinical note mentions: "Reviewed 3 wound photos from patient today (right heel, left ankle, sacral area)"
- Bill higher complexity code if multiple wounds justify it (99214 vs 99213)

### Q: "What if a payer requires modifier 95?"

**A:** Add it back selectively for that payer.

**Implementation:**
1. Add `payer` field to patients table
2. Create payer rules table:
   ```sql
   CREATE TABLE payer_rules (
     payer_name VARCHAR(100),
     requires_modifier_95 BOOLEAN DEFAULT FALSE
   );
   ```
3. Update billing logic:
   ```php
   $modifier = '';
   if ($patientPayer === 'BlueCross' && $payerRules['requires_modifier_95']) {
     $modifier = '95';
   }
   ```

### Q: "Can we still do synchronous video calls to use modifier 95?"

**A:** Yes! That would make modifier 95 appropriate.

**Hybrid Model:**
- Asynchronous photo reviews: No modifier, $92-180 per review
- Synchronous video calls: Modifier 95, potentially higher reimbursement

But this adds complexity (scheduling calls, coordinating patient availability). Only do this if:
- Payers explicitly require synchronous for telehealth
- You have staff to schedule/coordinate calls
- Patients are willing to schedule 10-15 min video calls

### Q: "What about CCM conflicts?"

**A:** Same provider can't bill both CCM and E/M for same patient same month.

**Solution:**
- Don't enroll wound photo patients in CCM
- OR: Bill wound reviews under different provider
- OR: Build conflict checker (recommended for future):
  ```php
  // Check if patient is enrolled in CCM with this provider
  $ccmCheck = $pdo->prepare("
    SELECT COUNT(*) FROM ccm_enrollment
    WHERE patient_id = ? AND provider_id = ? AND status = 'active'
  ");
  $ccmCheck->execute([$patientId, $userId]);

  if ($ccmCheck->fetchColumn() > 0) {
    // Show warning or block billing
    jerr('Patient enrolled in CCM. Cannot bill E/M same month.');
  }
  ```
